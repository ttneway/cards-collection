# 校園集卡牌 Backlog

這份文件是目前專案待辦事項的單一來源，依 `P1`、`P2`、`P3` 排序，同級內再依既定處理順序排列。

## P1-1 週期性任務異常
- 狀態：`done`
- 優先級：`P1`
- 問題描述：每日、每週、學期、自訂等週期性任務目前看起來沒有正確重置，像每日任務只能領取一次。
- 預期行為：任務應依設定週期正確重新開放領取，並在同一週期內遵守上限限制。
- 影響範圍：任務列表、掃碼領取、任務完成紀錄、週期 key 判定、任務 RPC。
- 備註 / 待確認：已改為由後端統一判斷當前週期，並修正 `period_key` 歧義錯誤；後續若有新邊界案例再另開項目。

## P1-2 掃碼免登入完成任務
- 狀態：`done`
- 優先級：`P1`
- 問題描述：目前學生完成任務仍綁定登入流程；需求是掃描身分條碼後，不需輸入帳號密碼、不需登入，即可完成任務。
- 預期行為：在符合任務規則的前提下，學生只需出示身分條碼，由掃描器掃描即可完成任務。
- 影響範圍：發點工作站、掃碼任務流程、身分條碼資料模型、任務完成紀錄、安全與權限邏輯。
- 備註 / 待確認：已實作公開 `/claim` 掃碼頁，流程為先掃任務條碼、再掃學生身分條碼，並由 Supabase 匿名 RPC 寫入紀錄。

## P1-3 多方式登入
- 狀態：`done`
- 優先級：`P1`
- 問題描述：登入頁需支援 `Email + 密碼`、`姓名 + 密碼`、`身分條碼 + 密碼`；目前程式碼已有部分實作，但仍需 migration 與線上驗證確認。
- 預期行為：三種登入方式都能穩定使用，且錯誤訊息清楚；姓名重複時應提供安全且明確的處理方式。
- 影響範圍：登入頁、auth store、Supabase RPC、profiles/scan_code 對應邏輯、GitHub Pages 線上驗證。
- 備註 / 待確認：`00011`、`00015` 已套用；Email、姓名、身分條碼登入皆可用，且姓名重複時會明確要求改用 Email。

## P2-1 任務編輯與刪除
- 狀態：`done`
- 優先級：`P2`
- 問題描述：教師後台目前可建立任務，但缺少編輯與刪除任務能力。
- 預期行為：教師能修改既有任務設定，也能刪除任務，並且刪除時對既有完成紀錄、條碼、工作站流程有明確處理。
- 影響範圍：教師任務管理頁、tasks schema 與刪除策略、任務條碼與紀錄查詢。
- 備註 / 待確認：已實作任務編輯與軟刪除（以停用方式保留任務條碼與歷史紀錄）；若未來需要硬刪除，再另外補安全限制。

## P2-2 任務增加下次可取得間隔時間
- 狀態：`todo`
- 優先級：`P2`
- 問題描述：目前任務只有週期與每週期上限，沒有同一任務兩次取得之間的冷卻時間設定。
- 預期行為：教師可設定同一任務的最短再次取得間隔，例如 30 分鐘、4 小時或 1 天。
- 影響範圍：任務 schema、教師任務設定頁、任務領取與掃碼 RPC、紀錄查詢。
- 備註 / 待確認：需決定冷卻時間與既有週期限制的優先順序與共同判斷方式。

## P2-3 管理者角色
- 狀態：`todo`
- 優先級：`P2`
- 問題描述：目前只有 `student`、`leader`、`teacher`，缺少可管理教師與高權限設定的管理者身分。
- 預期行為：新增 `admin` 或等效最高權限角色，可新增教師、管理教師與處理高權限設定。
- 影響範圍：profiles role、RLS policy、管理後台導覽、教師管理功能、登入後權限判斷。
- 備註 / 待確認：需盤點所有現有 `teacher` 專屬權限，決定哪些升級給 `admin`、哪些保留給 `teacher`。

## P3-1 教師後台未完成模組
- 狀態：`todo`
- 優先級：`P3`
- 問題描述：教師後台中的 `卡牌管理` 與 `管理成就` 目前沒有實作功能。
- 預期行為：教師可在後台管理卡牌資料與成就資料，包含建立、編輯、啟用/停用等核心操作。
- 影響範圍：教師後台頁面、cards/achievements schema、表單與列表 UI。
- 備註 / 待確認：需先定義第一階段最小可用功能，避免一次做太大。

## P3-2 卡片集分冊
- 狀態：`todo`
- 優先級：`P3`
- 問題描述：卡片集目前缺少「不同事件、不同本」的結構，無法清楚區分活動卡冊與常駐卡冊。
- 預期行為：卡片集可依事件或主題分冊，使用者能切換不同卡本查看與收集進度。
- 影響範圍：卡牌資料模型、卡牌圖鑑 UI、我的牌庫、卡包與活動內容配置。
- 備註 / 待確認：需決定分冊是獨立資料表、cards 新欄位，還是事件/系列抽象。

## P3-3 QR Code 辨識問題

- 優先級：`P3`
- 問題描述：目前部分 QR code 有無法辨識的情況，但原因尚未整理清楚。
