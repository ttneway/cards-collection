# 校園集卡牌 Backlog

目前已知問題與待辦，依優先級排序，並以這份文件作為唯一來源。

## ✅ 已完成

### P1-1 週期性任務異常
- 狀態：✅ 已完成
- 優先級：P1
- 問題描述：每日、每週、學期、自訂週期沒有正確重置，導致像每日任務只能領一次。
- 預期行為：每個任務都應依照自己的週期重新開放領取，並正確計算當期次數。
- 影響範圍：任務頁、掃碼發點 RPC、Supabase period key 計算。
- 備註 / 待確認：已修正 period key 與查詢邏輯，線上 migration 已套用。

### P1-2 掃碼免登入完成任務
- 狀態：✅ 已完成
- 優先級：P1
- 問題描述：使用掃描器掃學生身分條碼後，仍需要登入才能完成任務。
- 預期行為：在允許的任務情境下，學生可直接以任務碼 + 身分條碼完成任務，不需先登入。
- 影響範圍：公開掃碼頁、掃碼 RPC、學生條碼流程。
- 備註 / 待確認：已新增公開掃碼領取流程。

### P1-3 多方式登入
- 狀態：✅ 已完成
- 優先級：P1
- 問題描述：登入頁需同時支援 Email、姓名、身分條碼三種識別方式搭配密碼登入。
- 預期行為：使用者可以自由切換登入方式，並收到清楚錯誤提示。
- 影響範圍：登入頁、auth store、identifier lookup RPC、profiles / scan_code。
- 備註 / 待確認：migration 與前端已補齊，仍可持續做使用體驗微調。

### P2-1 任務編輯與刪除
- 狀態：✅ 已完成
- 優先級：P2
- 問題描述：教師後台只能建立任務，不能編輯或刪除。
- 預期行為：教師可修改既有任務，也可停用刪除任務，同時保留歷史紀錄。
- 影響範圍：任務管理頁、tasks schema、既有發點紀錄顯示。
- 備註 / 待確認：目前刪除為停用模式。

### P2-2 任務增加下次可取得間隔時間
- 狀態：✅ 已完成
- 優先級：P2
- 問題描述：除每週期上限外，還需要限制同一任務兩次取得之間的最短間隔。
- 預期行為：教師可設定冷卻分鐘數；學生或掃碼流程若尚未到時間，系統應拒絕並顯示提示。
- 影響範圍：tasks schema、任務管理頁、任務頁、掃碼 RPC、狀態 RPC。
- 備註 / 待確認：已新增 `claim_cooldown_minutes`，前後端同步接上。

### P2-3 管理者角色
- 狀態：✅ 已完成
- 優先級：P2
- 問題描述：目前只有 `student`、`leader`、`teacher`，缺少更高權限的管理者角色。
- 預期行為：新增 `admin` 角色，可管理教師、管理者與其他高權限帳號，並能接手教師後台所有權限。
- 影響範圍：profiles role、RLS policy、掃碼工作站、教師後台、帳號管理流程。
- 備註 / 待確認：已新增 `/admin` 管理頁；首次啟用時若系統尚無 admin，現有教師可自升為第一位管理者。

### P3-1 教師後台未完成模組
- 狀態：✅ 已完成
- 優先級：P3
- 問題描述：卡牌管理與成就管理頁目前尚無實作。
- 預期行為：教師可管理卡牌資料、卡池、成就條件與獎勵。
- 影響範圍：cards / achievements schema、教師後台 UI。
- 備註 / 待確認：已補上 `管理卡牌` 與 `管理成就` 的可用後台頁面，支援新增、編輯與上下架。
