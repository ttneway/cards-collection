## 待處理項目

### P3-2 卡片集分冊
- 狀態：待處理
- 優先級：P3
- 問題描述：卡片收集需要支援不同主題、不同冊本的結構，讓卡牌能依活動或系列分開收藏。
- 預期行為：學生可以切換不同收集冊；同一張卡只屬於單一本收集冊；若抽到重複卡，收集冊卡面可顯示持有張數。
- 影響範圍：卡片圖鑑、我的牌庫、卡片資料模型、冊本切換 UI。
- 備註 / 待確認：需要補 `album` 或 `series` 的明確資料結構與排序方式。

### P3-3 QR Code 辨識穩定性
- 狀態：待處理
- 優先級：P3
- 問題描述：部分 QR Code 在真實掃描器或手機鏡頭上有辨識失敗情況。
- 預期行為：任務碼、身分碼、功能碼都能穩定被常見掃描器辨識。
- 影響範圍：條碼列印頁、QR 生成尺寸、容錯等級、掃描流程。
- 備註 / 待確認：要區分是內容格式、輸出尺寸、對比度，還是掃描器相容性問題。

### P3-6 卡片等級中英文標示
- 狀態：待處理
- 優先級：P3
- 問題描述：卡片稀有度目前只顯示等級代號或中文其中一種，辨識度不夠一致。
- 預期行為：系統統一顯示像 `普通 (N)`、`稀有 (R)`、`超稀有 (SR)` 這類格式。
- 影響範圍：抽卡結果、卡片圖鑑、我的牌庫、商店、公告。
- 備註 / 待確認：資料層目前已有稀有度代號，可直接在前端顯示格式化標籤。

### P3-8 個人首頁最新公告
- 狀態：待處理
- 優先級：P3
- 問題描述：首頁目前缺少「系統最新公告」區塊，無法集中顯示新任務或管理公告。
- 預期行為：新增任務時可自動產生公告；管理者也可以在後台新增一般公告；學生首頁顯示最新公告清單。
- 影響範圍：首頁、管理者後台、公告資料表、任務建立流程。
- 備註 / 待確認：要先定義公告是否有置頂、截止日、可見角色等欄位。

### P3-9 新增卡牌收集冊
- 狀態：待處理
- 優先級：P3
- 問題描述：需要正式加入「收集冊」功能，讓不同主題的卡片收納在不同冊本裡。

### P3-10 AI 生成卡牌圖片
- 狀態：待處理
- 優先級：P3
- 問題描述：教師目前還不能直接輸入提示詞生成卡牌圖片。
