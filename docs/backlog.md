## 待處理清單

### P1-6 自主註冊帳號有身分碼但沒有 QR Code
- 狀態：待處理
- 優先級：P1
- 問題描述：目前自己註冊的帳號雖然已經有身分碼，但個人頁面沒有同步產生或顯示 QR Code，導致無法直接用掃描器使用。
- 預期行為：只要帳號擁有有效身分碼，就必須能在個人頁面顯示可掃描的 QR Code，並可列印或下載。
- 影響範圍：個人首頁、學生條碼頁、條碼元件、列印/下載流程、scan code 生成邏輯。
- 備註 / 待確認：需確認是否因角色、班級未歸類、或舊資料缺欄位造成 QR Code 未生成。

### P3-2 卡片集分冊
- 狀態：待處理
- 優先級：P3
- 問題描述：卡片集需要支援不同事件、不同主題的分冊結構，讓卡牌依照收集冊分類，而不是全部混在同一本。
- 預期行為：教師或管理者可以建立多本收集冊，每本收集冊有自己的主題、封面、卡牌列表與收集進度。
- 影響範圍：卡牌資料結構、卡片集 UI、卡牌歸屬、學生端收集進度顯示。
- 備註 / 待確認：可能需要新增 `album` 或 `series` 等資料模型。

### P3-3 QR Code 辨識穩定性
- 狀態：待處理
- 優先級：P3
- 問題描述：仍需持續確認哪些 QR Code 在不同掃描器上辨識率不足，並整理失敗情境。
- 預期行為：任務碼、身分碼、功能碼都能穩定被常見掃描器辨識。
- 影響範圍：條碼列印頁、QR 生成尺寸、容錯等級、掃描流程。
- 備註 / 待確認：需整理實機掃描失敗案例，確認是內容格式、圖片尺寸、留白，還是掃描器相容性問題。

### P3-6 卡片等級中英文標示
- 狀態：待處理
- 優先級：P3
- 問題描述：卡片等級目前只有中文，不利於抽卡機率公告與卡片資料一致性。
- 預期行為：卡片等級統一顯示為中英文格式，例如 `普通 (N)`、`稀有 (R)`、`超稀有 (SR)`。
- 影響範圍：卡牌資料、抽卡畫面、公告文字、卡片詳情。
- 備註 / 待確認：要先定義完整等級表，避免後續機率設定與顯示名稱不一致。

### P3-9 新增卡牌收集冊
- 狀態：待處理
- 優先級：P3
- 問題描述：需要新增真正的卡牌收集冊功能，每本收集冊包含不同卡牌，且不同收集冊內卡牌不重複。
- 預期行為：學生可以切換不同收集冊查看進度；若抽到相同卡牌，收集冊卡面可顯示持有數量。
- 影響範圍：卡牌庫、抽卡結果、持有數量、收集冊 UI。
- 備註 / 待確認：這項和 `P3-2` 高度相關，實作時可以合併規劃。

### P3-10 AI 生成卡牌圖片
- 狀態：待處理
- 優先級：P3
- 問題描述：希望教師能用提示詞快速生成卡牌圖片，但要維持統一風格，並與收集冊主題相符。
- 預期行為：教師輸入主題與提示詞後，可由 AI 生成卡圖，並套用到指定卡牌或收集冊。
- 影響範圍：教師後台、圖片生成流程、Storage、卡牌資料欄位。
- 備註 / 待確認：需要先定義風格模板、提示詞格式，以及圖片審核/替換流程。
